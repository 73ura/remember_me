# ブランチ戦略

### ブランチ命名規則

例：feat/timeline_rui

| プレフィックス | 意味 | どんなとき？ |
| --- | --- | --- |
| `feat` | 新機能追加 | ログイン機能を追加するとき |
| `fix` | バグ修正 | 合計金額が間違って計算されるバグを直すとき |
| `refactor` | リファクタリング（挙動は変えずコード整理） | 複雑な関数を整理する |
| `chore` | 雑務（依存更新、CI設定、ビルド関連） | Dockerfile修正、ライブラリアップデートなど |
| `docs` | ドキュメント関連 | README更新、仕様書追加など |
| `test` | テストコード関連 | 単体テストを追加・修正するとき |
| `style` | コードのスタイルや見た目 | Prettier整形、CSS微修正など |

### コミットコメント

| プレフィックス | 意味 | 例 |
| --- | --- | --- |
| `feat` | 新機能追加 | `feat: ログイン機能を追加` |
| `fix` | バグ修正 | `fix: 合計金額の計算ミスを修正` |
| `refactor` | リファクタリング（挙動は変えず整理） | `refactor: 認証処理を簡略化` |
| `chore` | 雑務（CI/CD, 依存更新, ビルド関連） | `chore: 依存ライブラリを更新` |
| `docs` | ドキュメント更新 | `docs: READMEにAPIの使い方を追記` |
| `test` | テスト関連 | `test: ユーザーサービスの単体テストを追加` |
| `style` | コードや見た目の修正（動作変化なし） | `style: Prettierでコードを整形` |
| `perf` | パフォーマンス改善 | `perf: 集計APIのクエリを最適化` |
| `build` | ビルドシステム関連（webpack, vite, docker など） | `build: Dockerfileのベースイメージを更新` |
| `ci` | CI/CD 関連 | `ci: GitHub ActionsにLintチェックを追加` |
