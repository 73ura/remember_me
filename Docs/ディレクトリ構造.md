# ディレクトリ構造ガイド

このドキュメントでは、`remember_me`プロジェクトのディレクトリ構造と各ディレクトリ・ファイルの役割について説明します。

## 📁 ルートディレクトリ構造

```
remember_me/
├── .git/                    # Gitリポジトリ情報
├── .github/                 # GitHub関連設定
├── backend/                 # Rails API バックエンド
├── frontend/                # Vue.js フロントエンド
├── Docs/                    # プロジェクトドキュメント
├── README.md                # プロジェクト概要
├── .gitignore              # Git除外設定（共通）
└── .gitattributes          # Git属性設定
```

---

## 📂 主要ディレクトリ

### `.github/`
GitHub Actions、Pull Requestテンプレート、IssueテンプレートなどのGitHub関連設定を格納します。

- `PULL_REQUEST_TEMPLATE.md` - PR作成時のテンプレート
- `PULL_REQUEST_REVIEW_TEMPLATE.md` - PRレビュー時のチェックリスト

**使い方:**
- PRを作成する際、自動的にテンプレートが表示されます
- レビュー時にチェックリストを確認して品質を保証します

---

### `backend/`
Rails APIサーバーのソースコードです。RESTful APIを提供し、フロントエンドからのリクエストを処理します。

#### 主要サブディレクトリ

**`app/`** - アプリケーションの主要コード
- `controllers/` - APIコントローラー（リクエスト処理）
- `models/` - データモデル（ActiveRecord）
- `views/` - ビュー（主にメーラー用）
- `jobs/` - バックグラウンドジョブ
- `mailers/` - メール送信処理

**`config/`** - アプリケーション設定
- `application.rb` - アプリケーション基本設定
- `routes.rb` - ルーティング定義
- `database.yml` - データベース接続設定
- `environments/` - 環境別設定（development/test/production）
- `initializers/` - 初期化処理

**`db/`** - データベース関連
- `migrate/` - マイグレーションファイル（スキーマ変更履歴）
- `schema.rb` - 現在のデータベーススキーマ
- `seeds.rb` - 初期データ投入スクリプト

**`lib/`** - ライブラリ・ユーティリティ
- `tasks/` - カスタムRakeタスク

**`test/` または `spec/`** - テストコード
- RSpecを使用する場合は`spec/`ディレクトリを使用

**`bin/`** - 実行可能スクリプト
- `rails` - Railsコマンド
- `rake` - Rakeタスク実行
- `setup` - 開発環境セットアップスクリプト

**`config.ru`** - Rackアプリケーション設定（Puma等のサーバー起動に使用）

**`Gemfile`** - Ruby依存関係の定義

**`Dockerfile`** - Dockerイメージの定義

**`docker-compose.yml`** - 開発環境用Docker Compose設定

**`.gitignore`** - バックエンド固有のGit除外設定

#### 一時ファイル・ディレクトリ（Git管理外）
以下のディレクトリは自動生成されるため、`.gitignore`で除外されています：

- `tmp/` - 一時ファイル、キャッシュ（bootsnap等）
- `log/` - ログファイル
- `node_modules/` - Node.js依存関係（Railsでも一部使用）
- `vendor/bundle/` - バンドル済みGem
- `storage/` - アップロードファイルの保存先（開発環境）

**使い方:**
```bash
# 開発サーバー起動
cd backend
bundle install
bin/rails server

# データベースマイグレーション
bin/rails db:migrate

# コンソール起動
bin/rails console
```

### `frontend/`
Vue.js 3を使用したフロントエンドアプリケーションです。Viteをビルドツールとして使用します。

#### 主要サブディレクトリ

**`src/`** - ソースコード
- `App.vue` - ルートコンポーネント
- `main.js` - アプリケーションエントリーポイント
- `assets/` - 静的ファイル（CSS、画像等）
  - `css/` - スタイルシート
- `components/` - Vueコンポーネント（予定）
- `views/` - ページコンポーネント（予定）
- `router/` - ルーティング設定（予定）
- `store/` - 状態管理（Pinia等）（予定）
- `api/` - API通信処理（予定）

**`public/`** - 静的ファイル（ビルド時にそのまま出力）

**`node_modules/`** - Node.js依存関係（Git管理外）

**`dist/`** - ビルド成果物（Git管理外、デプロイ時に生成）

**`package.json`** - Node.js依存関係の定義

**`vite.config.js`** - Viteビルド設定

**`tailwind.config.js`** - Tailwind CSS設定

**`postcss.config.js`** - PostCSS設定

**`.gitignore`** - フロントエンド固有のGit除外設定

**使い方:**
```bash
# 依存関係インストール
cd frontend
npm install

# 開発サーバー起動
npm run dev

# ビルド（本番用）
npm run build
```

## 🚫 Git管理外のディレクトリ・ファイル

以下のディレクトリ・ファイルは自動生成されるため、Gitで管理していません：

### バックエンド
- `backend/tmp/` - 一時ファイル・キャッシュ
- `backend/log/*.log` - ログファイル
- `backend/node_modules/` - Node.js依存関係
- `backend/vendor/bundle/` - バンドル済みGem
- `backend/storage/*` - アップロードファイル（開発環境）

### フロントエンド
- `frontend/node_modules/` - Node.js依存関係
- `frontend/dist/` - ビルド成果物
- `frontend/.vite/` - Viteキャッシュ
- `frontend/.cache/` - Parcelキャッシュ（使用する場合）

## 📝 開発時の注意事項

1. **依存関係のインストール**
   - バックエンド: `bundle install`
   - フロントエンド: `npm install`

2. **環境変数の設定**
   - `.env.example`があれば、`.env`としてコピーして設定
   - `.env`ファイルはGit管理しない

3. **データベースのセットアップ**
   - バックエンドで`bin/rails db:create db:migrate`を実行

4. **開発サーバーの起動**
   - バックエンド: `bin/rails server`（通常はポート3000）
   - フロントエンド: `npm run dev`（通常はポート5173）

5. **ビルド成果物**
   - `dist/`や`tmp/`は自動生成されるため、手動でコミットしない

---

## 🔄 ディレクトリ構造の変更時

1. 新しいディレクトリを追加する場合は、必要に応じて`.gitignore`を更新
2. 重要な設定ファイルは必ずGit管理
3. 一時ファイルやビルド成果物は`.gitignore`に追加
4. ディレクトリ構造の変更はこのドキュメントにも反映
